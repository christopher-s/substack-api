# Code of Conduct

## Purpose

This project exists to build useful software. We expect all participants
to conduct themselves professionally and to focus on the quality of the
work. This document defines the behavioral boundaries for participation.

## Standards

**Expected:**

- Write good code, submit clear issues, and provide honest review feedback
- Keep discussions technical and on-topic
- Accept technical criticism without taking it personally
- Be direct and substantive

**Prohibited:**

- Doxxing — publishing private personal information of any contributor
- Spam — off-topic promotion, astroturfing, or flooding issues/PRs
- Threats — credible threats of violence or legal action against contributors
- Sabotage — intentionally introducing bugs, breaking builds, or wasting reviewer time
- Fraud — misrepresenting authorship, test results, or benchmarks

## Enforcement

The maintainer(s) will judge violations based on observable behavior in
project spaces (issues, PRs, discussions, code). Decisions are based on
what happened, not who did it.

Consequences range from a warning to a permanent ban, at the maintainer's
discretion. There is no appeal process beyond asking the maintainer to
reconsider.

## Scope

This applies to project spaces on GitHub and any official project channels.
What contributors do elsewhere is their own business.

## No Ideological Tests

Contributors will never be asked to affirm, denounce, or participate in
any political, social, or ideological position as a condition of
participation. Technical merit and professional conduct are the only
criteria.
